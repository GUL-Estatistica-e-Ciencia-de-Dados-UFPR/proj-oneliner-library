Initialize git repository, stage all files and create empty .gitignore

Author: Marcos de Carvalho Date: 2026-04-25

Initializes a new git repository in current directory, stages all existing files, and creates an empty .gitignore file.

Language

git

Category

version-control

Command

git init && git add . && touch .gitignore

Explanation

This command sequence creates a new git repository (git init), adds all files in the current directory to the staging area (git add .), and creates an empty .gitignore file (touch .gitignore) for future ignore patterns.

Tags

git, version-control, repository, init, gitignore

Dependencies

git

Arguments

  1. DIRECTORY (Optional): Directory where to initialize repository
    Default: .

Examples

  1. git init && git add . && touch .gitignore - Initialize repository in current directory

  2. cd /path/to/project && git init && git add . && touch .gitignore - Initialize repository in specific directory

Output

Initialized empty Git repository in /home/user/project/.git/

Notes

  • The .gitignore file is empty after creation; you should add patterns to ignore unnecessary files

  • If .gitignore already exists, touch command updates its timestamp

  • Use git status to verify files are staged

Warnings

  • git add . stages all files including potentially sensitive data

  • Review staged files before committing
